# SherlockCite 中文实战指南

## 防止 AI 编造参考文献：按身份核对，而不是按搜索命中

SherlockCite 的公开 skill 本体使用英文，是为了兼容 Codex、Claude Code、Cursor 等 Agent Skills 生态；但中文论文核验是一等场景。中文参考文献的内容、证据摘录、人工判断说明，都可以用中文。

核心原则：

> 查不到不等于编造。只有确认“同一篇文献”才算核验通过。

## 状态映射

建议在 Excel/CSV 中使用英文状态，避免跨系统乱码；给中文读者解释时使用下面的中文含义。

| Status | 中文含义 | 使用条件 |
| --- | --- | --- |
| `VERIFIED` | 真实 | CNKI、万方、维普、期刊官网、学位库、出版社页等强来源确认同一篇；或两个以上独立来源交叉确认。 |
| `PLAUSIBLE` | 基本可信 | 百度学术、谷歌学术、作者单位、机构成果页、他文参考文献等能支持同一篇，但还缺强一手题录。 |
| `PENDING` | 待核 | 未查到、只找到相似线索、页面受限、字段不完整；不建议删除。 |
| `DUPLICATE` | 重复 | 同一篇文献重复出现。 |
| `SUSPECT` | 疑似编造/不采用 | 多源反查无记录，且存在明确冲突或异常，例如刊期不存在、题名作者来源互相矛盾、DOI 指向另一篇。 |

## 中文文献优先核验路径

### 中文期刊论文

1. CNKI 公开题录或机构入口
2. 万方
3. 维普
4. 期刊官网、期刊目录、采编平台
5. 百度学术
6. 作者单位官网、研究院成果页、教师主页
7. 其他论文参考文献中的逐字引用

### 中文学位论文

1. CNKI 博硕士库
2. 万方学位论文
3. 学校图书馆、机构知识库、学院公告
4. 百度学术
5. 导师或作者单位成果页

### 政策、报告、行业材料

1. 政府、协会、发布机构官网
2. 官方 PDF、新闻稿、公告页
3. 国家哲社、机构库、图书馆目录
4. 稳定二级来源

## 百度学术使用注意

中文文献常见情况：

- 直接用 `作者 + 完整题名` 可能返回 0。
- 只用“题名”更容易找到同题结果。
- 新版百度学术入口和旧版入口返回结果不同。
- 自动访问可能遇到安全验证；必要时用可见浏览器人工打开。

百度学术只能单独支持 `PLAUSIBLE`，通常不要单凭百度学术升为 `VERIFIED`。若百度学术结果显示同题、作者、来源、年份，建议再补 CNKI/万方/维普/期刊官网。

## 给朋友或导师看的说明

可以直接使用这段话：

> 本次核验采用“身份核对”标准，不以搜索引擎置顶结果作为真假依据。题名、作者、来源、年份等字段能对应到同一篇文献时，才判为真实或基本可信。检索不到的条目先标为待核，不直接判定为 AI 编造；只有出现明确著录冲突或多源反查均无记录并伴随异常时，才建议不采用。

## 推荐输出表头

工具层建议用英文表头：

```csv
id,reference,status,source_url,secondary_urls,screenshot_file,matched_fields,evidence_note,problem_note,next_action
```

内容可以写中文，例如：

```csv
30,"李梦涵.绿色信贷政策对污染企业投融资和创新的影响[D].南京大学,2021.",PLAUSIBLE,"https://xueshu.baidu.com/...","","ref-030.png","题名、作者、年份匹配","百度学术显示同题，作者李梦涵，学位，2021年；建议补 CNKI/万方学位库。","","保留，定稿前补强来源。"
```

## 最重要的反例

不要这样做：

- Tavily 第一次没搜到，就判 AI 编造。
- 搜索置顶结果题名相似，就判同一篇。
- 证据 URL 指向作者主页或无关 PDF，却把它当作该文献证据。
- 截图只截搜索结果列表，但没有显示题名、作者、来源、年份等身份字段。

正确做法：

- 先确认是否同一篇。
- 证据页面必须显示足够身份字段。
- 查不到先 `PENDING`。
- 宁可保守待核，也不要误删真实文献。
